# Dual-agent protocol translator — heterogeneous LLM verification turns natural-language ELISA into a real robotic Bradford assay

Paper link: https://arxiv.org/abs/2606.20120

## Script

Today's pick is a preprint from the Korea Institute of Machinery and Materials, posted to arXiv this week. The system has no catchy acronym — the authors just call it a dual-agent protocol translation framework — but it runs on a robotic platform they've named Bio Forge One, and the result is one of the more concrete bridges I've seen between natural-language biology protocols and a robot actually pipetting samples.

The setup. Biology protocols are written for humans. Lab robots speak in well coordinates, dispense volumes, incubation timers, and a strict execution graph. Closing that gap with a single large language model — feed it the protocol, get back machine commands — fails in instructive ways: it misroutes wells, drops parallel dispensing steps, swaps reagent identities. The interesting question is not whether an LLM can do this end to end, because it can't reliably, but what scaffolding around an LLM does the job. This paper proposes a scaffolding worth thinking about.

The architecture. Two agents. A parser agent takes the natural-language protocol — say, a Bradford assay or an ELISA — and rewrites it into a structured intermediate form, with explicit tags separating steps the human does at the bench from steps the robot will execute. A rule-based mapping engine, hard-coded to the robot's physical constraints, then turns that intermediate into device commands. So far this is sensible but not new. The novel piece is the second agent: a validation agent that reads back the generated commands and checks three things — that all the steps from the original protocol are present, that the parameters and unit conversions are right, and that the execution order makes physical sense. When it finds a problem, it issues structured feedback and the loop runs again, up to three attempts.

The mechanism intuition — and this is the part that's worth carrying with you. The validation agent is deliberately a different model family from the parser. The authors swept seven parsers against three validators — GPT five point oh, GPT four point one, the smaller GPT four point one mini and nano variants, the o-four-mini reasoning model, Llama four Maverick, Llama three point three seventy billion as parsers, paired with GPT five, Llama four Maverick, or Claude Sonnet four point six as validators. The sweep is the experiment. And it has a sharp answer. When the validator was Claude Sonnet four point six — a different model family than every parser tested — parameter accuracy climbed steeply across regeneration attempts for every parser. GPT four point one mini and Llama four Maverick went from roughly forty percent accuracy on the first attempt to seventy or eighty percent by attempt three. Even GPT four point one nano, the smallest parser, recovered to a cumulative pass rate near forty percent. When the validator was GPT five — same family as several of the parsers — accuracy barely moved from baseline. Llama four Maverick as validator, same story. The same-family validators were essentially nodding through their own mistakes.

So the headline finding is not that bigger models parse protocols better — they do, but that's predictable. The finding is that the model checking the work has to be a stranger to the model doing the work. Cross-family verification is what makes the self-correction loop functional, and intra-family verification is close to useless. If you're building any agent system with a checker stage — and a lot of the recent benchmark literature is converging on this pattern — pick your verifier from a different lineage than your producer.

The wet-lab demonstration. They closed the loop. GPT five as parser, Claude Sonnet four point six as validator, a Bradford protein quantification protocol fed in as plain English, output translated into commands for Bio Forge One, and the robot ran the assay. Calibration curve on bovine serum albumin from five to twenty-five micrograms per milliliter looked clean — optical density readings climbed monotonically from zero point five to about one point oh. Real fetal bovine serum samples came back at protein concentrations distinguishable across batches, in the low single-digit micrograms per milliliter range. Both the Bradford run and the parallel ELISA run passed validation on the first attempt — no regeneration needed for the strongest parser-validator pair.

What this doesn't yet do, to keep things honest. The rule-based mapping engine is bespoke to the KIMM platform — porting the system to a different robot means rewriting that engine. The protocols are microplate-based, which is the easy case structurally; arbitrary wet-lab procedures with custom glassware are not in scope. And the protocol library here is short — ELISA and Bradford, both very standardized assays. The interesting test would be a complex multi-day cell culture protocol with branching decisions based on intermediate readouts, and that's not what got demonstrated.

The Su Lab read. Two things to take from this. First, autonomous biology is going to be unlocked by agent-to-agent scaffolding far more than by single-model scaling — the protocol-to-robot bridge here works because of the loop, not because the parser is large. Second, heterogeneity is a structural property of working agent systems. If you're orchestrating LLMs in your own pipelines, the impulse to pick the cheapest model family for everything is exactly wrong for verification — that's where you want the foreign auditor.
