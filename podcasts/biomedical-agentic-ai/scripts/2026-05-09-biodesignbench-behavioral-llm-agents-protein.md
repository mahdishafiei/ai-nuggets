# 2026-05-09 — BioDesignBench: behavioral diagnosis of frontier LLM agents on protein design

Paper link: https://www.biorxiv.org/content/10.64898/2026.05.06.723381v1

## Script

Welcome back. This is the fifth nugget for Saturday, May 9, 2026 — and the last one. I'm shipping it because it's a piece that sits exactly at the intersection of two earlier episodes today, MP2D on the protein-design side and Open-Rosalind on the agent-benchmark side, and it gives a third data point that triangulates both.

The paper went up on bioRxiv yesterday, May 8th. It's called "Benchmarking and behavioral characterization of LLM agents for protein design," from Jiwoo Kim and Phil Romero at Duke. Phil Romero's lab has been doing protein machine learning for over a decade — directed evolution, sequence-function models, antibody engineering — so when his group runs a third-party evaluation of LLM agents on protein design tasks, that is worth paying attention to.

Here is what they did. They built BioDesignBench: seventy-six expert-curated protein design tasks spanning five categories — antibodies, enzymes, fluorescent proteins, binders, and protein scaffolds. Critically, the benchmark is not just a scoring function on outputs. It includes human baselines, non-LLM-agent baselines like deterministic hardcoded pipelines, and a set of behavioral metrics derived from the agent's tool-use trace — what the agent decided to do, in what order, and how it evaluated its own work. Then they run four frontier LLM agents through it and report both outcome scores and process scores.

The headline numbers are a mixed bag in a way that I think is genuinely useful. The strongest agents do beat deterministic hardcoded pipelines — that is, just running the standard tool chain in a fixed order is a lower bar than what a competent LLM agent can clear. So there is real value being added by the agent's ability to choose. But the same agents consistently underperform expert practice. The frontier agents are not yet at the level of a domain practitioner doing protein design by hand, even on tasks the practitioner has only seventy-six examples of. So the agents are above one floor and below one ceiling, which is the most informative possible position for a benchmark to land them in.

The behavioral diagnosis is where this paper earns its keep. The authors don't just report a score — they take apart the trace and identify three specific failure modes. First, agents generally do select appropriate tools — so the routing problem, the thing Open-Rosalind worried about, is largely solved at frontier scale for protein design. Second, the agents evaluate candidate designs far too shallowly — they call one or two scoring tools and accept the result, rather than running a full multi-metric battery and cross-checking. Third, they rarely compare alternatives side by side, and they terminate exploration prematurely. So we have a competent dispatcher with a lazy reviewer, sitting on top of a tool ecosystem that would actually support deeper analysis if asked.

The next part is the part that should change how you build agents this quarter. They run two interventions. First, guided workflows that prescribe which tools to call. That improves tool coverage, but it does not improve evaluation depth — the agent calls more tools and still glances at each result. Second, they explicitly enforce deeper multi-metric evaluation as part of the workflow — the agent must run multiple scorers and explicitly compare. That second intervention substantially improves agent performance. Their framing is the right one: the limitations are behavioral, not fundamental capability constraints. The model knows how to do the deeper analysis. It just won't unless you make it.

There is a tidy connection here to Open-Rosalind from earlier today. Both papers, working on totally different tasks — bioinformatics question answering versus protein design — converge on essentially the same operational claim. Free-form agentic reasoning underperforms structured workflow-constrained execution, and the structuring needs to be at the level of how the agent evaluates evidence, not just which tools it calls. That's two independent groups, two different domains, same conclusion in the same week. At this point I'd treat it as the working hypothesis for biomedical agent design.

A few open questions I'll flag. The four frontier agents aren't named in the abstract, so we don't know whether the result holds across vendors or whether one model is dragging the average. Seventy-six tasks is small for protein design — antibodies and enzymes are very different sub-problems, and the per-category accuracy probably matters more than the aggregate. And "expert practice" as a ceiling depends a lot on whose expert you ask. The full paper has the breakdowns. They've also released open-source reference agents and a public leaderboard, which is the right move — link in the show notes.
