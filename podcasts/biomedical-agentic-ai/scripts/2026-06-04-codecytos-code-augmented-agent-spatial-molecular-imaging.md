## Script

Today's pick is CodeCytos, a coding-based reasoning agent framework for spatial molecular imaging analysis from Hien Nguyen's group at the University of Houston, with co-authors at Houston Methodist. The first version posted to bioRxiv on May thirtieth and surfaced in our window yesterday.

Paper link: https://www.biorxiv.org/content/10.64898/2026.05.30.728935v1

The setup will feel familiar to anyone who's tried to use a conventional spatial-omics analysis tool for anything beyond the buttons the developer happened to put on the screen. Tools like the standard cell-segmentation-and-feature-extraction pipelines do the obvious things well — segment cells, compute morphology, report a fixed menu of neighborhood statistics — but the second a bioscientist asks a question that isn't on the menu, the workflow either stops or gets handed off to a bioinformatician who writes custom code. That gap, between the predefined feature catalogue and the open-ended question a pathologist actually wants to ask, is the whole reason these tools end up underused in practice.

CodeCytos takes the position that the agent's action space should be code itself, not a fixed set of button presses. The bioscientist asks a natural-language question — "what's the spatial relationship between tumor-infiltrating CD8 T cells and PD-L1-positive epithelium in this section, broken down by distance band?" — and a large language model with strong coding capability writes, runs, and iterates on the analysis code against the spatial molecular imaging data. The framework is evaluated under what the authors call a minimal prompt setting, meaning the user doesn't get to coach the agent with task-specific instructions or hand-crafted context. They just ask the question. The benchmark spans four expert-curated datasets across four tissue types — frontal cortex, non-small-cell lung cancer, pancreas, and tonsil — and they test it across multiple LLM backbones known for code generation, which is the right way to factor out "this only works on one specific model."

Here's the result worth sitting with for a minute. The authors show that giving the agent a handful of in-context coding-reasoning examples from completely outside the spatial-omics domain — that is, examples of code-reasoning behavior on unrelated tasks, randomly sampled — substantially improves performance on the spatial cellular feature task. They don't need to commission a domain expert to hand-craft in-domain demonstrations of "this is how you compute a neighborhood enrichment score." The general coding-reasoning competence transfers.

This is the load-bearing finding for anyone thinking about how to deploy code-generation agents on a new biomedical problem. The conventional fear is that you need expensive expert-crafted demonstrations to make few-shot in-context learning work for a specialized domain. CodeCytos says no, the bottleneck is general coding-reasoning quality, and the few-shot examples are doing something closer to "remind the model what step-by-step coding looks like" than "teach it the domain." If that holds up under wider testing, the cost curve for spinning up a competent code-as-action-space agent on a new spatial assay, or a new sequencing modality, or a new imaging platform, drops sharply. You don't need a panel of domain experts up front — you need access to the data, a decent prompt scaffold, and a coding-strong LLM backbone.

A few honest caveats. Four tissue datasets is enough to demonstrate breadth across organ systems, but it's not enough to characterize where this kind of agent fails — and code-as-action-space agents do fail, often quietly, by producing analyses that run cleanly and answer a question other than the one asked. The minimal-prompt evaluation is the right stress test for usability, but it also makes the ground truth harder to define — what counts as a correct answer to a vague natural-language question gets philosophical fast. And as always with this class of work, the comparison that would really land is a head-to-head against a competent computational biologist given the same questions and the same time budget. I'll wait to see how the v2 evaluation comes in.

The broader pattern here is the one to internalize. Across the agentic AI for biomedicine literature over the last six months, the most durable wins keep coming from systems that treat code as the action space and let the LLM compose analysis from primitives, rather than systems that try to wire the LLM into a fixed menu of pre-built analysis tools. The fixed-menu approach hits a ceiling exactly when the question gets interesting. CodeCytos is another data point on that curve, and the cross-domain few-shot transfer result is the part worth taking seriously for anyone designing the next agent in this space.
