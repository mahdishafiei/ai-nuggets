# EpiBench — verifiable benchmark shows no AI agent passes half of real epigenomics analysis tasks

Paper link: https://arxiv.org/abs/2606.13602

## Script

Welcome to Biomedical Agentic AI for Friday, June 13th, 2026.

Today's pick is a new benchmark from LatchBio called EpiBench, and the headline number is the kind that should make anyone building agentic systems for biology stop and think. Across sixteen model-harness pairs and over five thousand trajectories, not a single combination passed even a simple majority of the tasks. The best system, GPT-five-point-five running on the Pi terminal coding harness, landed at forty-five percent. The next tier — GPT-five-point-five with OpenAI's Codex harness, Claude Opus four-point-eight Max on Pi, and GPT-five-point-four on Pi — clustered between thirty-nine and forty percent, with confidence intervals that all overlap. The leaderboard isn't really a leaderboard yet. It's a band of frontier systems that all fall short of the same bar.

Here's what makes this evaluation worth paying attention to. EpiBench is a hundred and six tasks drawn from real epigenomics workflows: CUT-and-Tag and CUT-and-RUN on zebrafish chromatin, ATAC-seq and ChIP-seq from pediatric B-cell leukemia, and whole-genome bisulfite methylation from esophageal squamous cell carcinoma. Each task snapshots a realistic workflow state right before a decision point — quality control, peak calling, chromatin state comparison, differential methylation, downstream motif and gene-set analysis — and asks the agent to produce a structured answer that a deterministic grader can check exactly. Not "did the analysis look reasonable," not "did an LLM judge approve the writeup" — did the agent recover the same empirical result an experienced bioinformatician would, from the same files, in the same context.

That verifiability is the whole methodological intuition. The team explicitly designed the tasks to specify what should be recovered, not how. Most evaluations let the agent pick its own analysis path; the grader only cares whether the answer matches what the data supports. They went further and pruned tasks that could be solved by a generic biological prior alone, or by shortcuts that didn't actually require reading the files. The result is a benchmark that measures the thing we actually want to know — can the agent ground its conclusion in this specific assay's evidence — rather than measuring whether the agent has seen a similar workflow on the internet.

And the failure mode the data exposes is genuinely interesting. Component-level scoring — looking at every field the agent had to fill in — shows that sixty-eight percent of fields pass, while only thirty-one percent of endpoint answers pass. There's a big gap there. The agents are routinely finding the right files, running plausible tools, computing useful intermediate values, and then choosing the wrong final answer. The authors call this a separation between tool execution and scientific judgment, and the worked examples are concrete. In CUT-and-RUN spike-in normalization, agents picked end-to-end Bowtie2 alignment instead of local, which silently distorts the normalization downstream. In whole-genome bisulfite data, agents counted the two Bismark output rows for a CpG dinucleotide as independent sites, doubling coverage and shifting the differential-methylation effect sizes. In ATAC-seq, they handed paired-end BAMs straight to MACS3 with the paired-end flag, bypassing the single-base Tn5 cut-site representation that ATAC peak calling is supposed to use, and pushing every downstream motif call off-center.

The most striking observation in the paper is buried in the discussion. In manual review of GPT-five-point-five trajectories, the authors found cases where the agent had computed the correct answer, the right number was visible in its own scratch output, and then before submitting it the agent replaced the value with a more familiar workflow default or a literature-derived expectation. That's not a tool-use failure. That's the model overriding its own evidence with prior. It's exactly the failure mode you don't want from an autonomous analysis agent, and it suggests that scaling tool-calling capability without better grounding in the actual assay artifacts isn't going to close this gap.

A couple of caveats worth flagging. The task mix is uneven — CUT-and-Tag and CUT-and-RUN supply forty-seven of the hundred and six evaluations, ChIP-seq only ten, so the per-assay numbers are descriptive rather than controlled. Repeated failure modes recur across related tasks, which means aggregate scores partially reflect repeated tests of the same underlying judgment errors. And deterministic graders by construction don't capture every scientifically valid analysis path — they're a measurable sample, not a complete test of epigenomics reasoning. But the authors are honest about all of this in the methods, and even with the caveats, a verifiable benchmark with realistic file states and a thoughtful failure taxonomy is exactly what the field has been missing. The benchmark, public example evaluations, and representative trajectories are being released on GitHub.

The take here, for anyone building agentic systems on biomedical data: the bottleneck is no longer whether the agent can find the file or run the command. It's whether the agent will trust the file it just read over the workflow it has seen ten thousand times in pretraining. That's a grounding problem, not a tooling problem, and EpiBench gives us a way to measure progress on it. That's all for today. See you tomorrow.
